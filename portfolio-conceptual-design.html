portfolio conceptual design

Before building this site, I sketched a simple wireframe for how the portfolio should feel and function. The layout is not meant to be flashy — it is a frame for projects, learning notes, and documentation about how the site itself is built.

Sidebar

The left column is for identity and navigation. My name sits at the top, with links to About and my resume close by. The sidebar stays visible on every page so visitors always know where they are and can move between sections without hunting for a menu.

In the wireframe, a moon icon marks the top of the sidebar. That was a placeholder for a small visual anchor — something quiet that signals this is a personal site rather than a generic template.

Main project area

The large area on the right is where the work lives. Each project gets its own card with a title, short description, and a link to a dedicated page. The cards stack vertically so the page reads top to bottom like a list of work, not a grid of thumbnails competing for attention.

Below the projects, the same area holds blog and learning documentation — notes on flexbox, styling choices, and process. The main column is one continuous reading space for both finished work and work-in-progress thinking.

Footer

The wireframe includes a footer band for social links and resume access. The idea is to keep secondary links out of the main reading flow but still easy to find at the bottom of the page. Some of that information already lives in the sidebar on the live site; the footer is a backup place for the same essentials when the page gets longer.

What I kept from the sketch

The live site follows the wireframe’s core idea: a fixed sidebar plus a flexible main column built with flexbox. Projects and blog posts share the same card pattern. Typography stays minimal — Proxima Nova for headings, Courier for code and buttons — so the layout structure stays clear.
