What is a class? — A class is a label on an HTML element. CSS targets that label to apply styles. Example: .sidebar in styles.css styles every element with class="sidebar".

Live setup (5 CSS classes only)

Every portfolio page above links to styles.css. Layout is flexbox only — no CSS grid, no JavaScript, no mobile hamburger menu. Learning notes live on this page only; other pages are content + inline experiments.

Class Purpose
layout Flex row — sidebar + main (display: flex)

main flex: 1 content area

Inside a project card

Cards use one class on the outer element. Inner parts use plain HTML tags styled in CSS: 

<a class="card" href="...">
  <strong>Title</strong>     <!-- .card strong in styles.css -->
  <p>Subtitle</p>         <!-- .card p -->
  <img src="...">         <!-- .card img -->
  <span class="placeholder"> <!-- .card .placeholder -->
</a>

Modifier pattern: class="card" on the outer element plus class="placeholder" on the span when there is no screenshot. Placeholder styling comes from .card .placeholder in styles.css.

Inline styles

Most layout experiments bypass classes and go straight on the element: 

<div style="display: flex; justify-content: center;">
<p style="color: #999999; margin: 0 0 24px;">

Classes and inline styles can stack on the same tag, e.g. class="card" style="margin-bottom: 0;".

Classes vs IDs

Class (.name)
Reusable on many elements. Example: every page has class="layout".
ID (#name)
One per page. This simplified site uses few IDs; most layout is classes + inline styles.

How each page works

Page titles — Use a plain <h1> inside main. Style it in styles.css under .main h1 (font, size, margin). On other pages the <h1> is commented out; uncomment to show a title.
Home (index.html) — Three project cards in main, each an <a class="card">. Cards stack vertically via default block layout and margin-bottom: 2.5rem on .card. The page heading is commented out in the HTML. 
<main class="main">
  <a class="card" href="neolithia/">...</a>
  <a class="card" href="learning-to-build-use-flexbox.html">...</a>
  <a class="card" href="how-this-site-is-styled.html">...</a>
</main>
Try adding style="gap: 24px;" on a flex wrapper around the cards, or style="gap: 24px;" on .layout to space sidebar and main.
About (about.html) — Same layout / sidebar / main shell as home. The portrait sits centered above the text using inline flexbox (not a CSS class): 
<div style="display: flex; flex-direction: column; align-items: center; gap: 32px;">
  <figure style="width: 100%; max-width: 42rem;">...</figure>
  <div style="width: 100%;">...</div>
</div>
Change max-width on the figure to resize the portrait.
Learning to build use flexbox (learning-to-build-use-flexbox.html) — Each card sits in its own flex row. HTML comments label each card. Edit justify-content on that row to move the card left, center, or right: 
<!-- Card two — project two -->
<div style="display: flex; justify-content: center;">
  <div class="card" style="margin-bottom: 0; color: #999999;">...</div>
</div>
Card one uses justify-content: left, card two center, card three right. All three cards are placeholders (no links or screenshots). Prefer flex-start / flex-end in new code — they are the standard flex values for left and right.
Column layout (same page) — Three columns, three boxes each. All flexbox, inline on that page: 
<div style="display: flex; gap: 16px;">           <!-- row of columns -->
  <div style="display: flex; flex-direction: column; flex: 1; gap: 16px;">
    <div style="border: 1px solid #222; padding: 12px;">1</div>
  </div>
</div>
Try flex-direction: column on the outer div to stack columns vertically.
